# 图片回显体验优化功能说明

## 1. 变更背景
在原有发布流程中，封面图虽然支持素材库与本地上传，但用户在以下场景中缺乏稳定反馈：

- 自动选择封面时，不清楚是否选中、选中了哪张、来源是什么。
- 手动选择后，预览与状态文案不统一，容易产生“当前封面是否生效”的疑问。
- 关闭后重新打开发布弹窗，已选封面的上下文感知较弱。

本次优化目标是：**让封面图选择过程可见、可理解、可恢复**。

## 2. 功能目标
本次迭代聚焦“封面回显全链路”体验，重点目标：

1. 即时回显：自动或手动选择后，尽快看到封面预览。
2. 状态明确：通过状态文案区分“待选择 / 自动匹配中 / 成功 / 失败”。
3. 来源可解释：自动选中后告知来源（正文第一张图 / Markdown 元数据）。
4. 可恢复：弹窗重开后恢复已选封面信息，降低重复操作。

## 3. 实现概览

### 3.1 发布弹窗封面回显状态统一
在 `PublishModal` 中引入统一回显渲染能力：

- 新增状态渲染方法 `renderCoverPreview(...)`。
- 统一处理封面区的图片展示、状态样式、状态文案。
- 以状态驱动 UI，而非分散在多个分支里直接操作 DOM。

状态覆盖：

- `idle`：未选择封面，提示后续会自动匹配。
- `auto-detecting`：正在自动匹配封面。
- `success`：封面匹配/选择成功（包含来源信息）。
- `error`：自动匹配或发布过程出现可恢复错误。

### 3.2 自动封面来源与预览增强
自动选择逻辑保持原优先级不变：

1. 正文第一张图
2. Markdown frontmatter（`cover / cover_image / thumbnail / thumb / image`）
3. 失败后提示手动上传

在自动命中时增加预览 URL 解析：

- `http`/`data:image`：直接回显。
- 本地路径：尝试解析 Vault 文件并转换为可预览资源路径。

### 3.3 手动选择回显增强
手动选择（素材库/本地上传）后：

- 立即在发布弹窗更新预览图。
- 状态文案明确标记为“已手动选择封面”。
- 与自动选择状态体系保持一致，不再出现“图有了但语义不清晰”。

### 3.4 重开弹窗恢复
发布弹窗初始化时读取 `sessionStorage.selected_material`：

- 若存在有效 `media_id`，直接恢复回显与状态文案。
- 减少二次打开时的重复选择成本。

### 3.5 素材库选中态优化
封面选择弹窗中，素材卡片增强选中视觉：

- 高亮边框与内阴影增强已选可见性。
- 支持根据已缓存 `media_id` 自动回显当前选中卡片。

## 4. 交互文案优化
本次对发布流程相关文案进行了统一与明确化：

- 自动匹配中：`正在自动匹配封面图...`
- 自动匹配失败：`自动匹配封面失败，请手动上传封面图`
- 手动成功：`已手动选择封面（手动选择）`
- 自动成功：`已自动选择封面（正文第一张图）` / `已自动选择封面（Markdown 元数据）`

## 5. 样式变更
新增/增强了以下视觉状态类：

- `.cover-preview-idle`
- `.cover-preview-loading`
- `.cover-preview-success`
- `.cover-preview-error`
- `.cover-status-text`
- `.material-item-selected`（增强选中视觉）

目标：让用户一眼区分当前封面状态，而不仅依赖 Notice 提示。

## 6. 验证结果
已执行构建验证：

- `npm run build`：通过（TypeScript 类型检查 + 生产构建）。

建议手动验证场景：

1. 正文含图，未手动选择：应自动选中正文首图并回显。
2. 正文无图但 frontmatter 有图：应自动选中 frontmatter 图并回显来源。
3. 自动匹配失败：应显示错误状态并可手动选择后继续发布。
4. 素材库手动选择：返回发布弹窗后应立即显示手动封面回显。
5. 弹窗关闭后重开：应恢复已选封面状态。

## 7. 兼容性与影响范围
- 不改变微信发布 API 调用协议。
- 不改变封面选择优先级策略。
- 主要影响发布弹窗与封面选择弹窗交互层，不影响文章正文转换逻辑。

## 8. 后续可选优化
1. 在封面选择弹窗中增加“当前已选”固定信息条，减少翻页后迷失。
2. 为自动选择失败提供更细粒度原因（无图、路径解析失败、上传失败）。
3. 增加“重试自动匹配”按钮，减少必须进入手动流程的频率。
